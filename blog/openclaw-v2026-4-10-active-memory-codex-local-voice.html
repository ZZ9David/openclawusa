OpenClaw v2026.4.10: Active Memory, Native Codex, and Local Voice — What It All Means

By OpenClaw USA Team April 13, 2026 5 min read

Just days after the headline-grabbing "Dreaming" feature landed in v2026.4.9, the OpenClaw team has shipped another major release. Version 2026.4.10 introduces three capabilities that, taken together, mark a significant step toward making AI agents feel less like chatbots and more like genuine personal assistants: the Active Memory plugin, native Codex model support, and local voice synthesis on Mac.

Here is what each feature does, why it matters, and what US users should keep in mind before flipping the switch.

Active Memory: Your Agent Finally Remembers You

The biggest addition in this release is the optional Active Memory plugin. In previous versions, OpenClaw could store memories — but only when you explicitly told it to. If you forgot to say "remember that I prefer bullet-point summaries" or "save my flight details," the context was gone by the next conversation.

Active Memory changes that dynamic entirely. When enabled, it inserts a dedicated memory sub-agent that runs right before the main reply. This sub-agent automatically searches your stored preferences, past decisions, and conversation history, then feeds the most relevant details into the response — all without you having to ask.

In practical terms, this means your OpenClaw agent can recall that you like morning briefings in a certain format, that your team uses Slack instead of Teams, or that you already discussed a particular project last Tuesday. The result is fewer "sorry, remind me again" moments and more fluid, human-like interactions.

The plugin ships with three context modes — message, recent, and full — so you can control how deeply the sub-agent digs into your history. There is also a /verbose flag for real-time inspection of what the memory layer is pulling in, which is a welcome transparency feature for privacy-conscious users.

By default, Active Memory runs in direct-message sessions but stays off in group chats and channels unless you opt in. That is a sensible choice: in a group setting, surfacing one person's private preferences could get awkward fast.

Native Codex Support: One Less Setup Headache

OpenClaw v2026.4.10 also adds a built-in provider for OpenAI's Codex and GPT-family models. Previously, connecting Codex required a community plugin or manual API configuration. Now it is a first-class option in the model selector, which means fewer steps for users who want to pair OpenClaw's agent framework with OpenAI's reasoning models.

For US users, this is especially relevant because Codex remains one of the most popular model choices for code-generation and technical workflows. Having it baked in reduces the friction of getting started and lowers the chance of misconfigured API keys or plugin version mismatches — two of the most common complaints on the OpenClaw community forums.

Local Voice on Mac via MLX

The third notable addition is experimental support for local voice synthesis through Apple's MLX framework on macOS. OpenClaw's Talk Mode has historically relied on cloud-based text-to-speech services, which introduces latency and raises privacy questions about voice data leaving your machine.

With the new MLX integration, Talk Mode can now run speech synthesis entirely on-device using Apple silicon. The feature is labeled experimental, so expect rough edges — but the direction is clear. Local inference means faster responses, no cloud dependency, and complete privacy for voice interactions.

This matters in the US market particularly for enterprise and healthcare users who face strict data-handling requirements. An AI agent that can listen, think, and speak without any data leaving the laptop is a much easier sell to compliance teams.

Security Hardening: SSRF and Execution Fixes

Alongside the headline features, this release includes hardened protections against server-side request forgery (SSRF) and node execution injection — two vulnerability classes that made headlines in OpenClaw's security crisis last month. The team has continued to tighten input validation and sandbox boundaries, which is reassuring given the nine CVEs disclosed in late March.

If you have been holding off on updating due to security concerns, this release and the rapid patch cadence since then should offer some confidence that the team is treating security as a top priority.

Should You Update?

For most users, yes. The Active Memory plugin alone is reason enough — it is the kind of quality-of-life improvement that makes daily use noticeably smoother. The Codex integration removes a common setup pain point, and the security fixes address real vulnerabilities.

The one caveat is that Active Memory is still new, and its behavior in complex, multi-agent workflows has not been extensively tested by the community yet. If you run OpenClaw in a production environment, consider enabling Active Memory in a test channel first and using the /verbose flag to audit what it retrieves before rolling it out widely.

To update, run npm update -g openclaw or pull the latest Docker image. The Active Memory plugin can be toggled on in your agent's configuration file under the plugins section.
