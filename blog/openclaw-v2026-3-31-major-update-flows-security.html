OpenClaw v2026.3.31: The Biggest Update Yet — Unified Flows, Tighter Security, and More

By OpenClaw USA Team April 1, 2026 5 min read

OpenClaw just released version v2026.3.31, and this is not a routine patch. Dropped on March 31, 2026, this update touches the very core of how OpenClaw manages tasks, enforces security, and connects to the platforms you use every day. If you run OpenClaw — or you’re thinking about setting it up — here’s what you need to know.

What Changed and Why It Matters

The headline feature in v2026.3.31 is something the OpenClaw team calls Unified Flows. Until now, if you had a scheduled cron job, a background sub-agent, and a command-line task all running at the same time, each one had its own lifecycle and its own way of being managed. That made it difficult to see what your AI agent was actually doing at any given moment.

With this release, all of those different task types are brought together under a single system backed by a local SQLite database. You can now run openclaw flows list, openclaw flows show, or openclaw flows cancel to see and control everything your agent is working on from one place. Each “Flow” is composed of individual “Tasks,” giving you a clear picture of what’s running, what’s waiting, and what’s finished.

For everyday users, this means less guesswork. If something seems stuck, you can check the status and cancel it. If a task fails, the system now surfaces recovery hints so you know what went wrong and how to fix it.

Security Gets a Major Overhaul

After the widely reported wave of nine CVEs disclosed in just four days back in mid-March, it’s no surprise that security is front and center in this release. The update follows one guiding principle: make implicit trust explicit. In practical terms, that means OpenClaw will no longer quietly assume that something is safe just because it’s running on your local machine.

Here are the most important security changes:

  • Plugin installs now fail by default if a critical security finding is detected. You have to explicitly opt in with a special flag to override this — no more silent risk.
  • Gateway authentication is stricter. The old behavior of trusting any connection from the same machine is gone. Local connections now require a configured token.
  • Node commands require pairing approval. Simply pairing a device is no longer enough to expose commands — an additional approval step is now required.
  • High-risk shell tools like awk and sed are excluded from the fast-path trust system, meaning they’ll always require explicit approval before your agent can use them.

These changes may require some adjustments to existing setups, but they directly address the kinds of vulnerabilities that made headlines in March. If you’re running OpenClaw for anything beyond casual experimentation, these guardrails are worth the minor inconvenience.

New Platform Support: QQ Bot, LINE Media, Matrix Upgrades

OpenClaw continues to expand the number of platforms it can interact with. The v2026.3.31 release adds a QQ Bot channel plugin — a big deal for users who communicate through Tencent’s QQ messenger, which is massively popular in Asia and increasingly relevant for cross-border teams.

The QQ Bot integration supports multi-account setup, slash commands, reminders, and full media send/receive. LINE also gets an upgrade with outbound media support for images, video, and audio. And Matrix — the open-source messaging protocol — gains several improvements including optional room history context, draft streaming for partial reply updates, and per-DM thread isolation.

Slack and WhatsApp also received targeted improvements. Slack now has native in-channel approval routing for exec commands, and WhatsApp added emoji reaction support for incoming messages. Microsoft Teams picks up Graph-backed member info lookups.

Smarter AI and Remote MCP Support

On the AI side, this release introduces remote HTTP/SSE server support for MCP (Model Context Protocol) configurations. This means you can now connect OpenClaw to remote tool servers with authentication headers — expanding what your agent can do without running everything locally.

There’s also a new /btw side question feature for Anthropic models that lets you ask a quick question during a session without interrupting the agent’s main reasoning flow. And cross-agent memory search now supports extra collections, so agents can draw on sessions from other agents when looking for relevant context.

Android Gets Smarter Notifications

If you run OpenClaw with an Android device in the loop, this update adds notification forwarding with package filtering, quiet hours, and rate limiting. That means your agent can monitor your phone’s notifications and act on them — but only the ones you choose, and only during hours you set. It’s a thoughtful balance between automation and not being overwhelmed.

What You Should Do Now

If you’re already running OpenClaw: Update carefully. This release includes several breaking changes, especially around gateway authentication and plugin installs. Review the official release notes before upgrading, and test your existing flows after updating.

If you’re new to OpenClaw, this is actually a good time to start. The security improvements in v2026.3.31 make the platform meaningfully safer than it was even two weeks ago, and the unified Flows system makes it much easier to understand what your agent is doing.

With over 50 contributors and continued momentum — the project has surpassed 135,000 GitHub stars — OpenClaw is evolving fast. This release shows a team that’s listening to the security community and building the management tools that a maturing platform needs.
