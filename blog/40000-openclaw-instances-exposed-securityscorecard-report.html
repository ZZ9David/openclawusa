40,000 OpenClaw Instances Exposed Online: What US Users Need to Do Right Now

April 23, 2026 OpenClaw USA Team 5 min read

A fresh security report this week puts a sharp number on a worry many OpenClaw users have quietly had for months: how many of those "personal AI agents" running at home and at work are actually sitting wide open on the public internet? According to cybersecurity firm SecurityScorecard, a big one — and it lands just as OpenClaw's popularity in the US is accelerating.

Researchers say they found 40,214 OpenClaw instances reachable from the internet, across 28,663 unique IP addresses hosting OpenClaw control panels. Roughly 63% are vulnerable to remote code execution, meaning a bad actor could take full control of the machine the agent is running on — often without the owner ever clicking a thing.

The short version: If you run OpenClaw and you have not updated it this month, or you made its web panel reachable from outside your home network, stop and patch before you do anything else today.

What the report actually found

The research focuses on OpenClaw instances where the management panel — the page where you log in to add skills, connect channels, and watch the agent work — is exposed to the open internet rather than kept behind a firewall or a VPN. Three separate high-severity CVEs are involved, with CVSS scores between 7.8 and 8.8. One of them, CVE-2026-25253, was disclosed back in February and allows one-click remote code execution if an attacker can lure a user to a malicious link while their agent is running.

What makes this disclosure more serious than the usual vulnerability note is that working exploit code is already circulating publicly. That lowers the bar dramatically — an attacker no longer needs deep skills to take advantage of an unpatched instance, only the IP address of an exposed panel.

Why this matters for US users

OpenClaw's appeal in the US has always been that it puts a real AI agent on your own hardware rather than in someone else's cloud. That autonomy is great for privacy and cost, but it shifts the security responsibility onto the individual. A hosted service has a team patching servers on your behalf; a self-hosted agent does not.

Several factors make the American install base especially exposed:

  • Home office deployments. Many US users run OpenClaw on a spare Mac Mini or home server and forward a port so they can use it on the go. Port forwarding without authentication in front of it is the single most common way these instances end up in scans.
  • Small-business pilots. Teams experimenting with OpenClaw for email triage and scheduling sometimes skip VPN setup because it feels too technical for early testing.
  • Always-on personal agents. Unlike a chat app you open and close, a personal agent is designed to run 24/7. That means it is online and reachable every hour of every day.

What an attack actually looks like

An attacker with an exposed panel's address could push code that runs with the same permissions as the user account hosting the agent. In practical terms, that could mean reading any file the agent can read — emails, browser cookies, notes, anything in connected workspace folders — and, in the worst cases, using the machine as a pivot point into the rest of a home or office network. Because OpenClaw agents often hold credentials for messaging apps, calendars, and note-taking tools, a single compromise can cascade well beyond one device.

What to do today

The good news is that protecting yourself from this cluster of issues is mostly free and takes a few minutes. The OpenClaw team has issued patches, and all three CVEs are covered in the current stable release.

  • Update to the latest version. Run the built-in update command or reinstall from the official download. v2026.4.9 added SSRF and node execution hardening after January's ClawHavoc incident; v2026.4.20 adds further fixes.
  • Take the panel off the public internet. If you do not need remote access, disable port forwarding. If you do, put the panel behind a Tailscale or WireGuard VPN rather than exposing it directly.
  • Turn on password and two-factor authentication on the panel itself, even at home. Default-open panels are the biggest single risk factor in the SecurityScorecard data.
  • Rotate any tokens your agent has stored — email, chat, calendar. If the machine has been online and unpatched for a while, assume those secrets could have leaked.
  • Scan your own IP with a free tool like Shodan's self-check. If your panel shows up publicly, a stranger has already seen it too.

The bigger picture

This is the second major security headline for OpenClaw in 2026 after January's ClawHavoc disclosures, and it lands while the project is adding features at a rapid clip. That tension — rapid feature growth versus a rapidly expanding attack surface — is now the defining story of the agentic AI category in the US. NVIDIA's NemoClaw early preview, which bolts policy-based guardrails onto OpenClaw, is one response; VPN-first guidance and stricter installer defaults are another.

For US users, the practical takeaway is simple: treat your OpenClaw instance the way you would a security camera or router — patch it, do not leave it open to the internet, and keep its secrets on a short leash. The convenience is real. So is the exposure.
